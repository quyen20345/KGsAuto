# AGENTS.md

Ngắn gọn repo guidance cho OpenCode. Tin CLI/package scripts/compose hơn prose nếu lệch.

## Shape
- Flow: markdown -> LLM extraction -> entity resolution -> Neo4j import -> Graph API/Chat API/frontend.
- APIs: `apps.graph_api.main:app` port `8000`; `apps.chat_api.main:app` port `8002`.
- `apps/chat_api` là chatbot API hiện tại; đừng revive `apps/rag_api`. Có OpenAI-compatible `/v1/models`, `/v1/chat/completions`.
- Frontend: `apps/frontend` (Vite/React); API calls ở `src/services/api.js`; env: `VITE_GRAPH_API_BASE_URL`/`VITE_API_BASE_URL`, `VITE_CHAT_API_BASE_URL`.
- `data/` chứa generated/extracted/indexed artifacts; tránh commit output lớn.

## Setup / run
```bash
pip install -r requirements.txt
pip install -e . --no-deps
docker compose up -d                         # infra only: Neo4j, Qdrant, Ollama
uvicorn apps.graph_api.main:app --host 0.0.0.0 --port 8000 --reload
uvicorn apps.chat_api.main:app --host 0.0.0.0 --port 8002 --reload
cd apps/frontend && npm install && npm run dev
```
- Copy `.env.example` -> `.env`; key vars: `OPENAI_COMPATIBLE_*`, `LLM_PROVIDER`/`LLM_MODEL`, `NEO4J_*`, `QDRANT_URL`, `RAG_*`, embedding vars.
- Compose mounts Neo4j/Qdrant under `${Browser}/...`; set `Browser` before relying on persisted data. Compose Neo4j auth: `neo4j/12345678`.

## Pipeline commands dễ nhầm
```bash
# extraction: default data/raw/uet -> data/extracted; skips existing unless --no-skip-existing
python -m services.extraction.cli --input-dir data/raw/uet --output-dir data/extracted --provider OpenAICompatible --model cx/gpt-5.3-codex

# entity resolution: memory backend phải chạy all trong 1 process (vectors ephemeral)
python -m services.entity_resolution.cli --stage all --input-dir data/extracted --store-backend memory --run-id demo_run

# staged/persistent: dùng qdrant và giữ nguyên run-id
python -m services.entity_resolution.cli --stage stage1 --input-dir data/extracted --store-backend qdrant --run-id my_run
python -m services.entity_resolution.cli --stage stage2 --store-backend qdrant --run-id my_run
python -m services.entity_resolution.cli --stage stage3 --store-backend qdrant --run-id my_run

python -m services.neo4j_import.import_to_neo4j --dir data/entity_resolution/artifacts/final/stage3/output_graph
```

## RAG / graph
- Public modes exactly: `semantic_search`, `graph_search`, `naive_grag`, `hybrid`.
- `semantic_search`: Qdrant + collection/index; graph modes: Neo4j + imported graph; `hybrid`: both.
- Runtime: `services/rag_system/pipeline.py`, wrapped by `apps/chat_api/main.py`; modes in `services/rag_system/modes/`.
- Use LLM via `services.llms.get_llm(...)`; no ad-hoc provider clients.
- Neo4j driver setup: `apps/graph_api/neo4j.py`; Qdrant used separately by entity resolution and RAG retrieval.

## Merge / conventions
- Online merge: `POST /api/entity/merge` rewires live Neo4j graph.
- Offline merge: entity-resolution stage3 rewrites JSON before import.
- Do not commit `.env`, secrets, logs, generated data, large artifacts.
- Commit messages: no AI attribution (`Co-Authored-By: Claude`, `Generated by Claude`, etc.).
- Under `thesis/`, follow `thesis/AGENTS.md`; treat as research material.
