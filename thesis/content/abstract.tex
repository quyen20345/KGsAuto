    \textbf{TÓM TẮT}

\vspace{1cm}

Các hệ thống hỏi đáp dựa trên mô hình ngôn ngữ lớn (LLM) đã được ứng dụng rộng rãi, nhưng thường gặp khó khăn khi trả lời câu hỏi dựa trên dữ liệu nội bộ, đặc biệt là các câu hỏi yêu cầu hiểu quan hệ giữa nhiều thực thể và truy vết nguồn gốc thông tin. Retrieval-Augmented Generation (RAG) kết hợp với Knowledge Graph là một hướng tiếp cận giúp hạn chế nguy cơ hallucination và tăng khả năng truy vết bằng chứng. Tuy nhiên, việc xây dựng Knowledge Graph tự động từ dữ liệu văn bản tiếng Việt và tích hợp vào RAG vẫn còn nhiều vấn đề mở, từ trích xuất, xử lý trùng lặp thực thể đến thiết kế chế độ truy xuất phù hợp.

Khóa luận này trình bày hệ thống KGsAuto, một pipeline xây dựng và khai thác Knowledge Graph từ dữ liệu Markdown về Trường Đại học Công nghệ. Hệ thống gồm bốn thành phần chính: (1) trích xuất thực thể và quan hệ bằng LLM với prompt có ràng buộc schema cho mô hình Property Graph, (2) pipeline Entity Resolution ba giai đoạn kết hợp chuẩn hóa, embedding, blocking, clustering và LLM-based matching để giảm trùng lặp thực thể, (3) lưu trữ Knowledge Graph trong Neo4j và lập chỉ mục văn bản trong Qdrant, và (4) hệ thống hỏi đáp đa chế độ gồm Semantic Search, Naive Graph RAG, Graph Search và Hybrid RAG.

Hệ thống được triển khai với Python, FastAPI cho các API (Chat API và Graph API), React/Vite cho frontend, Neo4j làm graph database và Qdrant làm vector database. Phần thực nghiệm báo cáo thống kê vận hành cho trích xuất và Entity Resolution do chưa có ground truth phù hợp, đồng thời đánh giá hệ thống hỏi đáp bằng RAGAS trên bốn chế độ truy xuất. Kết quả trên tập pilot 30 câu hỏi cho thấy Hybrid RAG đạt chất lượng tốt nhất với faithfulness 0.730 và context recall 0.623, trong khi Semantic Search là baseline nhanh nhất với độ trễ trung bình 6.54 giây mỗi câu. Các chế độ graph-only có context recall thấp hơn, cho thấy Knowledge Graph phát huy hiệu quả rõ nhất khi được kết hợp với bằng chứng văn bản trong Hybrid RAG.
